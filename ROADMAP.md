# Roadmap

## Principles

- Keep SQL visible.
- Keep args bound.
- Keep identifiers explicit.
- Avoid ORM behavior.
- Avoid code generation.
- Prefer small, boring APIs.

## Near-term

- Expand examples.
- Harden dialect tests.
- Improve docs.
- Improve scan and Codex documentation.
- Add compatibility notes.

## Not Planned

- Entity tracking.
- Relationship loading.
- Schema migrations.
- SQL generation from structs.
- Replacing `sqlc`.
- Hiding SQL behind repository magic.
